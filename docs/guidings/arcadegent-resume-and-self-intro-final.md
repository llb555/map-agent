## 1. 最推荐的项目定位

如果面试官问“这是什么项目”，最推荐你这样定义：

> 这是一个基于 LLM、Agent、RAG、FAISS 和地图服务的垂直场景 AI 应用，面向音游机厅搜索、附近推荐和路线建议场景，支持多轮对话、工具调用、知识检索和流式执行过程展示。

如果你想说得更工程一点，可以这样说：

> 这是一个前后端分离的大模型应用项目。后端基于 FastAPI 和 ReAct 风格运行时实现 Agent 编排、工具调用、会话状态管理和 SSE 推流，前端基于 React 和 Zustand 实现对话交互、历史会话、地图联动和执行过程可视化。

## 6. 三分钟自我介绍成品稿

这版适合技术一面、项目深聊开场。

> 我最近重点准备的是大模型应用开发方向，想找的是 Agent、RAG、向量检索、智能问答这类偏工程落地的岗位。  
> 我有一个比较能代表这个方向的项目叫 Arcadegent，它本质上是一个垂直场景 AI 应用，场景是音游机厅搜索、附近推荐和路线规划。用户可以通过自然语言提问，系统会调用不同的 worker 和工具来完成检索、地理解析、路线建议，然后通过 SSE 把执行过程和结果实时推回前端。  
> 在技术上，它不是简单地调一次模型 API，而是有 main agent、search worker、navigation worker、tool registry、session store、context builder 和 provider adapter 这些模块。我重点理解并梳理了从前端提交问题，到后端创建 session，再到 Agent 调度工具、更新 working memory、最后把结果通过 SSE 回推前端的完整链路。  
> 另外我还做了一个比较有代表性的能力扩展，就是图片输入。原来系统虽然支持上传图片，但模型并没有真正看到图片内容。我补齐了图片进入模型消息的多模态传递链路，随后在联调中发现当前 DeepSeek 接口并不支持 `image_url` 格式，所以我又加了一层视觉 provider 自动分流和友好降级提示。这个过程让我对 provider 差异、能力边界和应用层兼容设计有了更具体的理解。  
> 所以如果用一句话总结，我比较适合的是大模型应用工程、Agent 编排、RAG、工具调用以及多模态接入这类方向。

## 7. 项目介绍成品稿

如果面试官只问：

> 你介绍一下这个项目

你可以直接用这版。

> Arcadegent 是一个面向音游机厅搜索、附近推荐和路线规划的垂直场景 AI 应用。  
> 它的后端基于 FastAPI，内部实现了 ReAct 风格的 Agent 运行时、工具注册表、会话状态管理和 SSE 流式事件推送；前端基于 React 和 Zustand，负责对话交互、历史会话、地图联动和机厅浏览器。  
> 和普通聊天机器人不一样的地方在于，它不是只返回一段文本，而是会先识别意图，再调用搜索、地理解析、路线规划等工具，把结构化结果写入 working memory。知识库部分会先做文档切分和向量化，并写入 FAISS 索引，再基于相似度检索相关片段参与回答，最后把中间执行过程和地图 artifacts 逐步推回前端。  
> 这个项目比较能体现我在大模型应用工程上的能力，包括多轮对话、Agent 编排、RAG、FAISS 向量索引、工具调用、Provider 适配以及前后端联动。

## 8. 如果被问“你具体做了什么”

这是很高频的问题，下面给你两个版本。

### 8.1 稳妥版

> 我主要做的是本地复现、核心链路阅读和功能改造。  
> 一方面我把前端提交请求、后端创建会话、Agent 调度 worker、工具执行、working memory 回填和 SSE 推流这条链路完整梳理了一遍；另一方面我做了一些更贴近真实应用的问题修复和能力补齐，比如图片输入的多模态链路和 provider 兼容处理。  

### 8.2 更强一点的表达

> 我在这个项目里主要做了两类事情。  
> 第一类是系统级理解和链路梳理，包括 Agent 编排、工具调用、会话状态、SSE 事件流和前后端联动。  
> 第二类是能力扩展和兼容性改造，比如补齐图片输入到模型的消息链路，增加视觉 provider 分流，以及把不支持多模态的底层报错转换成用户可理解的降级提示。

## 11. 最后提醒

这个项目最适合帮你证明的是：

- 大模型应用工程能力
- Agent 编排能力
- RAG 和对话系统能力
- 向量检索与 FAISS 索引能力
- 工程化和前后端联动能力

它不适合硬包装成：

- 模型训练平台
- 底层推理优化平台
- 大规模分布式 AI 基础设施

最稳的策略永远是：

- 把做过的讲具体
- 把没做过的讲边界
- 把理解到的讲清楚

这样反而最像一个靠谱的候选人。
